# 考研健身记录系统 · 手机端同步指南

本文档说明如何将当前仅在本机运行的健身追踪应用，部署到公网并适配手机端使用，实现电脑与手机的数据同步。

---

## 一、总体思路

当前应用是纯前端单页应用（HTML + JS + Tailwind CSS），数据同步依赖 Supabase。要上手机端，只需解决一个问题：**让手机浏览器能访问到这个网页**。

有三种方案，按推荐程度排列：

| 方案 | 难度 | 费用 | 推荐度 | 适用场景 |
|------|------|------|--------|----------|
| **GitHub Pages 部署** | ⭐ 极低 | 免费 | ⭐⭐⭐ 首选 | 个人使用，零成本 |
| **Vercel / Netlify 部署** | ⭐ 低 | 免费 | ⭐⭐⭐ 首选 | 个人使用，自动 HTTPS |
| **自有服务器 / NAS** | ⭐⭐⭐ 高 | 有 | ⭐⭐ 备选 | 已有服务器或内网穿透条件 |

以下以 **GitHub Pages** 和 **Vercel** 为例，给出完整步骤。

---

## 二、方案 A：GitHub Pages 部署（推荐，完全免费）

### 步骤 1：创建 GitHub 仓库

1. 登录 [github.com](https://github.com)，点击右上角 **+ → New repository**
2. 仓库名填写 `fitness-tracker`（或其他名字）
3. 选择 **Public**（Private 也能用 Pages，但需 Pro）
4. 点击 **Create repository**

### 步骤 2：上传代码

```bash
# 在你的 fitness-tracker 文件夹中执行
cd /path/to/fitness-tracker

git init
git add index.html app.js README.md
git commit -m "init"

# 将下面命令中的 YOUR_USERNAME 替换为你的 GitHub 用户名
git remote add origin https://github.com/YOUR_USERNAME/fitness-tracker.git
git branch -M main
git push -u origin main
```

> 如果没有安装 Git，也可以直接在 GitHub 网页上点击 **Add file → Upload files**，把 `index.html` 和 `app.js` 拖进去上传。

### 步骤 3：开启 GitHub Pages

1. 进入仓库主页，点击 **Settings**
2. 左侧菜单找到 **Pages**
3. **Source** 选择 **Deploy from a branch**
4. **Branch** 选择 `main` / `root`，点击 **Save**
5. 等待 1-2 分钟，刷新页面，会看到提示：
   > Your site is live at `https://YOUR_USERNAME.github.io/fitness-tracker/`

### 步骤 4：手机访问

1. 手机浏览器（Safari / Chrome）打开上述链接
2. 用同一个 Supabase 账号登录
3. 数据自动从云端同步到手机

---

## 三、方案 B：Vercel 部署（推荐，自动 HTTPS + 国内访问稍快）

### 步骤 1：准备代码

先把代码推送到 GitHub（同方案 A 的步骤 1-2）。

### 步骤 2：连接 Vercel

1. 登录 [vercel.com](https://vercel.com)，用 GitHub 账号一键登录
2. 点击 **Add New Project**
3. 选择你的 `fitness-tracker` 仓库，点击 **Import**
4. **Framework Preset** 选择 **Other**
5. 点击 **Deploy**

约 30 秒后部署完成，Vercel 会自动分配一个域名：
`https://fitness-tracker-xxx.vercel.app`

### 步骤 3：自定义域名（可选）

Vercel 支持绑定自己的域名：
1. Vercel 项目页 → **Settings → Domains**
2. 输入你的域名（如 `tracker.yourdomain.com`）
3. 按提示在域名 DNS 添加一条 CNAME 记录指向 `cname.vercel-dns.com`
4. 等待 DNS 生效（通常几分钟）

## 四、Supabase 跨设备同步说明

无论用哪种部署方案，**数据同步的核心都是 Supabase**。请确保手机端和电脑端使用同一个账号登录。

### 同步机制

| 数据类型 | 存储位置 | 跨设备同步方式 |
|----------|----------|----------------|
| 日程记录（checkins） | Supabase `daily_checkins` 表 | ✅ 自动同步 |
| 训练记录（workouts） | Supabase `workout_logs` 表 | ✅ 自动同步 |
| 体重/腰围 | Supabase `weight_logs` 表 | ✅ 自动同步 |
| 用户配置（profile） | Supabase `profiles` 表 | ✅ 自动同步 |
| 体型照片 | `localStorage`（`ft_body_photos`） | ❌ **仅本地，不跨设备** |
| 饮水记录 | `localStorage`（`ft_local_water_*`） | ❌ 本地模式才用，登录后存云端 |
| 饮食记录 | `localStorage`（`ft_local_diet_*`） | ❌ 同上 |

### 首次在手机端使用的正确姿势

1. **电脑端**：确保已登录 Supabase，数据已上传云端（正常使用即可，数据会自动写入 Supabase）
2. **手机端**：打开部署好的链接 → 点击顶部「登录以同步」→ 用**同一个邮箱和密码**登录
3. 等待 1-2 秒，页面自动刷新，数据即从云端拉取到手机

> ⚠️ **重要**：如果之前在电脑端使用的是「本地模式」（未登录），本地数据**不会自动迁移**到云端。需要先在电脑端登录，数据才会写入 Supabase。

## 五、PWA 适配（添加到手机主屏幕，像原生 App 一样使用）

目前应用缺少 PWA 支持（README 中已标记为待办 #7）。以下给出最小化适配步骤，让手机可以「添加到主屏幕」。

### 步骤 1：创建 `manifest.json`

在项目根目录新建 `manifest.json`：

```json
{
  "name": "考研健身记录",
  "short_name": "健身记录",
  "description": "考研期间健身与日程追踪",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#0b0f19",
  "theme_color": "#10b981",
  "icons": [
    {
      "src": "icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}
```

### 步骤 2：创建图标（偷懒方案）

没有设计图标的话，可以用在线工具 30 秒生成：

1. 打开 [favicon.io](https://favicon.io/) 或 [maskable.app](https://maskable.app/)
2. 输入文字「💪」或上传一张简单图片
3. 下载 192×192 和 512×512 的 PNG，分别命名为 `icon-192.png` 和 `icon-512.png`
4. 放到项目根目录

### 步骤 3：创建 Service Worker（极简离线缓存）

新建 `sw.js`：

```javascript
const CACHE_NAME = 'fitness-tracker-v1';
const URLS_TO_CACHE = [
  '/',
  '/index.html',
  '/app.js',
  'https://cdn.tailwindcss.com',
  'https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2'
];

self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME).then(cache => cache.addAll(URLS_TO_CACHE))
  );
});

self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request).then(response => {
      return response || fetch(event.request);
    })
  );
});
```

### 步骤 4：在 `index.html` 中引入

在 `<head>` 内添加：

```html
<link rel="manifest" href="manifest.json">
<meta name="theme-color" content="#10b981">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
```

在 `<body>` 最底部、`app.js` 引入之前添加：

```html
<script>
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('sw.js');
  }
</script>
```

### 步骤 5：重新部署

把新增的 `manifest.json`、`sw.js`、两个图标文件一起提交并推送，等待自动部署完成。

### 步骤 6：添加到主屏幕

**iPhone（Safari）**：
1. 用 Safari 打开部署好的链接
2. 点击底部分享按钮（⬆️）
3. 选择「添加到主屏幕」
4. 桌面会出现 App 图标，打开即为全屏模式

**Android（Chrome）**：
1. 用 Chrome 打开部署好的链接
2. 点击右上角菜单（⋯）
3. 选择「添加到主屏幕」或「安装应用」
4. 桌面/应用抽屉会出现图标

## 六、手机端显示优化（已完成的响应式适配）

当前应用使用了 Tailwind CSS，本身具备基础响应式能力。以下列出已适配和可能需要微调的点：

### ✅ 已自然适配

- 所有卡片使用 `w-full` + `max-w` 限制，在手机屏上自动撑满
- Tab 导航按钮使用 flex 等分，小屏不会溢出
- 日程时间轴使用百分比定位，任意宽度可用
- Canvas 图表已做 DPR 适配和 resize 监听

### ⚠️ 建议进一步微调（可选）

| 位置 | 现象 | 建议 |
|------|------|------|
| 顶部 Header 按钮群 | 小屏可能换行拥挤 | 可把「登录」「退出」收到一个下拉菜单里 |
| 日期选择器 + 箭头 + 日报按钮 | 一行可能放不下 | 小屏下让「日报」按钮单独一行 |
| 训练页动作卡片 | 每组输入框横向排列 | 小屏改为纵向堆叠（重量/次数/RPE 各占一行） |
| 饮食页宏量统计 | 四个数字横向 | 小屏改为 2×2 网格 |
| 输入框字体大小 | iOS 会强制放大 `< 16px` 的输入框 | 把所有 `input` 的 `text-sm` 改为 `text-base` |

如需微调，在 `index.html` 的 `<style>` 中添加媒体查询即可：

```css
@media (max-width: 640px) {
  /* 小屏专用样式 */
  .header-btn-group { flex-wrap: wrap; gap: 4px; }
  .exercise-input-row { flex-direction: column; }
}
```

## 七、常见问题

### Q1：手机打开后白屏/黑屏？
- 检查 Supabase CDN 是否被墙：`https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2` 在国内可能加载慢，可替换为 `https://unpkg.com/@supabase/supabase-js@2`
- 检查浏览器控制台是否有 SyntaxError（某些旧版手机浏览器不支持 `?.` 可选链）

### Q2：电脑登录了，手机登录后数据没同步？
- 确认两端登录的是**同一个邮箱**
- 检查电脑端是否使用的是「本地模式」（未登录），本地数据不会自动上云

### Q3：体型照片能在手机上看吗？
- 不能。体型照片存在 `localStorage` 中，不上传 Supabase，因此不跨设备。
- 如需跨设备查看体型照片，可考虑：① 手动把照片发到微信/相册 ② 或后续改用 Supabase Storage 存储图片。

### Q4：没有 GitHub 账号，也不想学部署？
- 把 `index.html` 和 `app.js` 通过微信文件传输助手发到手机
- 手机用浏览器打开本地文件（iOS 用「文件」App → 分享 → 用 Chrome/Safari 打开）
- ⚠️ 这种方式**只能本地使用**，Supabase 同步功能因跨域限制可能无法正常工作，不推荐。

2. **手机 Safari/Chrome 打开 Pages 链接** → 用 Supabase 账号登录（2 分钟）
3. **可选**：创建 `manifest.json` + `sw.js` → 重新部署 → 添加到主屏幕（15 分钟）
