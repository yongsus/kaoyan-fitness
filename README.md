# 健身追踪系统

一个单页响应式暗色主题健身追踪应用，纯前端实现（Vanilla JS + Tailwind CSS），支持浏览器直接打开使用。

---

## 技术栈

| 层 | 技术 |
|---|---|
| UI | Tailwind CSS CDN、自定义暗色主题 |
| 前端 | Vanilla JavaScript（无框架） |
| 数据存储 | 优先 Supabase（PostgreSQL），fallback 到 localStorage |
| 图表 | 原生 Canvas 2D + SVG（无图表库依赖） |

---

## 全部功能一览

### 1. 用户与认证
- **Supabase 登录**：邮箱密码注册/登录
- **匿名登录**：无需邮箱快速体验
- **本地模式**：完全不联网，全部数据存 localStorage（`user_id: 'local'`）
- **登出/切换账号**：支持清除本地缓存后重新登录

### 2. 周期管理
- **4天训练循环**：上肢A → 下肢B → 上肢B → 下肢A，第5-7天为休息日
- **自定义起始日**：首次打开可选择"今天从第几天开始"
- **周/天编辑器**：随时调整当前周数和天数，或重置周期起始日为今天
- **自动推进**：每天打开自动顺延；若漏练可选择"继续"或"推进"

### 3. 日程追踪（核心计时器）
- **多分类计时**：学习、训练、饮食、放松、其他
- **学习子分类**：数学、408、政治、英语
- **智能推荐**：根据当前时间段自动推荐最合适的分类（00:00-23:59 全覆盖）
- **计时器防丢失**：刷新页面后自动恢复正在进行的计时
- **感受记录**：计时结束后可选记录感受（状态、效率、备注）
- **历史补签**：支持任意日期补录日程数据

### 4. 饮食追踪
- **固定菜单 + 勾选制**：早餐、上午加餐、午餐配菜、下午加餐、晚餐配菜、睡前，每项可独立勾选
- **蛋白质下拉选择**：午餐/晚餐各有 7 种蛋白质来源可选（鱼、鸡胸、牛排、虾等）
- **碳水下拉选择**：午餐/晚餐各有 5 种碳水来源可选（土豆、馒头、玉米、米饭、挂面），训练日/休息日自动切换推荐分量
- **休息日自动隐藏**：休息日自动隐藏上午加餐和橄榄油选项
- **饮水追踪**：+300ml / +500ml / 重置，饮水日志记录每次时间和总量
- **热量与宏量统计**：实时计算总热量、碳水、蛋白质、脂肪，带进度条和百分比
- **智能提示**：根据当前摄入与目标的差距，显示"维持期：热量平衡，力量优先"等提示；赤字过大/盈余过高时警告
- **饮食目标自定义**：默认可配置为 2600kcal / 320g碳水 / 160g蛋白质 / 70g脂肪（维持/重组模式）

### 5. 代谢计算
- **Mifflin-St Jeor 公式**：基于性别、年龄、身高、体重计算基础代谢
- **活动系数**：久坐(1.2) / 轻度(1.375) / 中度(1.55) / 高度(1.725) / 极高度(1.9)
- **热量策略**：维持 TDEE / 减脂 -200 kcal / 增肌 +250 kcal
- **动态显示**：结果区自动标注"（维持 TDEE）"/"（+盈余）"/"（赤字）"
- **数据持久化**：身体数据 localStorage 持久化

### 6. 训练追踪
- **4个训练日，共25个动作**：
  - 上肢A（7动作）：平板杠铃卧推、上斜哑铃卧推、坐姿哑铃推举、双杠臂屈伸、绳索下压、哑铃侧平举、杠铃弯举
  - 下肢B（5动作）：杠铃深蹲、罗马尼亚硬拉、腿举、腿屈伸、俯卧腿弯举
  - 上肢B（7动作）：引体向上、杠铃划船、哑铃划船、哑铃卧推、高位下拉、锤式弯举、农夫行走
  - 下肢A（6动作）：传统硬拉、前蹲、腿举、腿弯举、站姿提踵、坐姿提踵
- **热身模块**：每个训练日有专属热身动作，可逐项勾选完成
- **动作参数统一标注**：`[组数]组 x [次数] | 组间休息：[X]秒`
- **逐组记录**：每组可记录重量(kg)、次数/RPE/秒数、完成状态
- **特殊动作类型**：
  - `time` 类型（面拉等）：只记秒数，不计入容量
  - `carry` 类型（农夫行走）：只记秒数，不记重量/RPE，60秒组间休息
- **组间休息倒计时**：完成一组后自动弹出倒计时，带环形进度条；支持 ±15秒、暂停、跳过
- **渐进超负荷提示**：根据该动作历史最佳记录，提示"上次 Xkg，建议尝试 Ykg"
- **动作历史图表**：点击动作名查看该动作近几次训练的重量/容量趋势折线图
- **训练总结**：训练结束后显示总时长、总容量、各动作明细、感受统计

### 7. 统计与可视化
- **快速卡片**：本周完成率、连续打卡天数、本周学习时长、本周训练容量
- **今日时间分配**：SVG 环形图（学习/训练/饮食/放松/其他），hover 显示详细时长
- **本周回顾**：Canvas 柱状图（学习/训练/休息每日对比）+ 文字汇总
- **本月概览**：Canvas 折线图（每日学习时长趋势）+ 文字汇总
- **全部数据汇总**：总学习时长、总训练次数、总训练容量、全部体重变化折线图
- **本周打卡日历**：7天网格显示每日完成状态

### 8. 体重与腰围追踪
- **每周一记录**：体重输入区仅在周一显示（可通过日期选择器任意日期补录）
- **体重趋势图**：近8条记录的 Canvas 折线图
- **腰围独立存储**：与体重分开 localStorage 存储
- **体重统计**：总变化、平均值、趋势箭头

### 9. 放松记录
- **单人/双人记录**：底部 `···` 入口，全屏面板
- **快速记录**：一键标记"放松一次"
- **备注**：可添加文字备注
- **统计**：今日/本周/本月/全部次数
- **7天柱状图**：近7天放松频率可视化
- **数据隔离**：存 `schedule_data.relaxation`，不参与日程/统计/日报/饼图

### 10. 日报自动生成
- **每日 22:00 自动弹出**：时间线 + 时间分配饼图 + 关键数字 + 备注汇总
- **支持"今日不再弹出"**：勾选后当日不再自动触发
- **手动生成**：日程页和统计页均有"📋 日报"按钮
- **导出**：支持复制文字、保存为图片（html2canvas）

### 11. 周报自动生成
- **每周日 22:00 自动弹出**：本周 vs 上周对比（学习时长、训练次数、体重变化）
- **4周趋势迷你图**：近4周学习时长横向柱状图
- **下周预告**：自动识别是否为 Deload 减载周（每4周一次）
- **支持"本周不再弹出"**
- **手动生成**：统计页"📊 周报"按钮
- **导出**：支持复制文字、保存为图片

### 12. 数据备份与恢复
- **一键导出全部数据**：合并 Supabase 四张表（daily_checkins / workout_logs / weight_logs / profiles）+ 所有 `ft_*` localStorage 项为统一 JSON
- **从文件导入恢复**：上传备份 JSON，校验 `exportVersion` 和 `userId`，恢复 localStorage 后自动刷新页面
- **月末备份提醒**：每月最后一天自动显示横幅提示

### 13. 体型拍照记录
- **每月1号自动提醒**：横幅提示拍摄正面/侧面/背面照片
- **Canvas 压缩**：上传图片宽度压至 800px，JPEG 质量 0.7，单张 <500KB
- **三角度记录**：正面、侧面、背面独立上传和预览
- **历史时间轴**：按日期浏览以往所有体型照片
- **仅本地存储**：`ft_body_photos` localStorage，不上传云端
- **批量删除**：支持删除全部体型照片

### 14. 其他功能
- **每日首次打开弹窗**：提示"继续当前计划"或"推进一天"
- **日期历史浏览**：← → 箭头浏览任意日期，点击"返回今天"回到当前
- **开发者工具**：生成7天测试数据（含体重、日程、饮食、训练）、清除全部本地测试数据

## 已知限制与待办事项

### 🔴 高优先级

| # | 问题 | 说明 |
|---|---|---|
| 1 | **导入时 Supabase 数据无法真正恢复** | 导出的 JSON 包含 Supabase 表数据，但导入功能目前只恢复 localStorage。Supabase 数据的覆盖/合并/跳过策略尚未实现。 |
| 2 | ~~localStorage 5MB 容量风险~~ | ✅ 已解决：体型照片自动保留最近 6 个月，超过 3MB 时自动删除最旧月份。 |

### 🟡 中优先级

| # | 问题 | 说明 |
|---|---|---|
| 4 | **周报/日报图片导出依赖 html2canvas** | 当前未引入 html2canvas CDN，导出图片功能会降级为"复制文字"。如需图片导出，需引入 `<script src="https://cdnjs...html2canvas.min.js">`。 |
| 5 | **周报数据可更丰富** | 目前周报仅聚合学习时长、训练次数/容量、体重三项。可扩展：① 每日详细时间线 ② 饮食达标率 ③ 训练动作进步榜单 ④ 休息日/训练日比例。 |
| 6 | **放松记录未参与统计汇总** | 放松数据完全隔离，不进入任何统计。可考虑在统计页增加"本周放松次数"卡片，或纳入时间分配饼图。 |
| 7 | ~~无 PWA 支持~~ | ✅ 已解决：已添加 manifest.json + Service Worker，支持离线使用和添加到主屏幕。 |
| 8 | **无数据迁移工具** | 用户从本地模式切换到 Supabase 登录后，历史数据不会自动迁移。 |

### 🟢 低优先级 / 体验优化

| # | 问题 | 说明 |
|---|---|---|
| 9 | **训练感受只保存文本，未结构化** | 训练整体感受和单动作感受均为纯文本，无法用于后续分析（如"这周轻松的动作有哪些"）。 |
| 10 | **饮水记录不支持删除单条** | 只能重置全部饮水，不能删除某一条误录。 |
| 11 | **饮食页碳水选择无搜索/筛选** | 下拉选项较多时体验一般。 |
| 12 | **周/天编辑器无日期选择器** | 只能通过按钮增减，不能直接选某月某日作为起始日。 |
| 13 | **没有成就/徽章系统** | 连续打卡、容量突破、学习时长里程碑等可设计徽章激励。 |
| 14 | **没有多设备同步冲突处理** | 同一账号在多个设备使用时，Supabase 数据可能产生冲突（以最后写入为准，无版本控制）。 |
| 15 | **没有数据定期自动备份** | 目前只有手动导出，可考虑每周自动触发一次导出并提示下载。 |

```
├── workout.js         # 训练记录、容量计算、组间休息（~1000 行）
├── stats.js           # 统计图表、体重腰围、放松记录（~860 行）

├── service-worker.js  # PWA Service Worker（离线缓存）
└── README.md          # 本文档
```

## 本地使用方式

直接用浏览器打开 `index.html` 即可。首次使用选择"本地模式"，所有数据保存在浏览器 localStorage 中。

### PWA 安装（推荐手机使用）

1. 用浏览器（Chrome/Edge/Safari）打开 `index.html`
2. 地址栏会出现"添加到主屏幕"或"安装应用"提示
3. 安装后可在桌面/主屏幕像原生 App 一样启动，支持离线使用

### Supabase 云端同步

1. 在配置页填写 Supabase Project URL 和 Anon Key
2. 注册/登录账号
3. 数据会自动同步到 Supabase
